Style-Up Fullstack E-Commerce

A fully functional responsive shopping application containing search catalog query routers, state-managed dynamic shopping carts, and secure mock payment checkout flows.

Overview

Style-Up is a full-stack e-commerce web application designed to simulate a modern apparel boutique. It integrates client-side page layouts with an Express middleware backend and a MongoDB document database. Users can browse products, filter by categories, add items to a persistent cart, and finalize orders.

Need / Motivation

The goal was to build a secure, responsive, and robust web architecture that handles state, sessions, and catalog queries gracefully. Creating Style-Up provided a strong sandbox to master MVC patterns, routing, document schema design, and asynchronous database interactions.

Objectives

  • Deliver an interactive, responsive user interface with vanilla CSS styling.
  • Create structured API endpoints handling catalog retrieval, product additions, and shopping cart updates.
  • Design flexible MongoDB collections for products, customers, and order summaries.
  • Enforce secure session tokens on user state transitions and checkout routes.

System Architecture

Data flow within Style-Up. User triggers interaction -> Frontend makes API fetch request -> Express validates session -> Reads/Writes MongoDB catalog -> Requests mock transaction settlement.

Key Features & Implementation

  • Dynamic Search: Implements server-side regex searches on MongoDB to return instant, matching catalog lists as users query the shop.
  • Persistent Cart: Encrypts shopping session cookies, keeping item arrays saved in user carts even after page reloads.
  • NoSQL Schemas: Models relational concepts (e.g. Orders containing multiple products with specific quantities) using flexible nested documents in MongoDB collections.

Results

Style-Up demonstrates a solid, robust MVC implementation. The platform loads quickly, serves dynamic product updates instantly from MongoDB, and supports secure session boundaries between visitors and registered users during the ordering process.
